# 项目规划

## 1. 技术路线

MVP 可以先做 Web 端。

推荐技术栈：

- 前端：Next.js + React + Tailwind CSS
- 组件：shadcn/ui
- 编辑器：TipTap
- 后端：NestJS 或 FastAPI
- 数据库：PostgreSQL
- ORM：Prisma
- AI：OpenAI API 结构化输出
- PDF：Playwright HTML 转 PDF
- 文件存储：Supabase Storage 或 Cloudflare R2

当前目录里的第一版实现采用静态 HTML/CSS/JS，无需安装依赖，适合快速看产品方向和交互闭环。

## 5. 风险和对策

| 风险 | 影响 | 对策 |
| --- | --- | --- |
| AI 编造经历 | 影响可信度 | 只允许基于资料库生成，标记无依据内容 |
| PDF 样式不稳定 | 影响交付 | 使用固定 HTML 模板和 Playwright |
| 用户资料填写成本高 | 影响激活 | 支持从旧简历解析导入 |
| JD 格式差异大 | 解析不稳定 | 使用结构化输出和兜底规则 |
| 简历表达太 AI 味 | 降低真实感 | 提供语气选择和人工编辑 |

## 6. 首批验收标准

- 用户可以粘贴 JD 并看到结构化岗位画像
- 用户可以导入旧简历并生成资料库草稿
- 用户可以看到匹配度和优化建议
- 用户可以生成一份与 JD 相关的简历
- 用户可以查看关键简历表达的事实依据
- 用户可以导出 PDF
- 用户可以保存投递记录
- 用户可以记录一次面试复盘，并反哺简历优化
