# 墨刀原型说明

说明：当前环境不能直接登录或操作墨刀账号，因此这里提供一份可以直接复制到墨刀中搭建的产品原型稿。页面结构、组件、文案、视觉规范都已经整理好。实现项目里的 `index.html` 也同步做成了可交互原型，可以作为墨刀视觉参考。

## 1. 设计风格

不要使用科技蓝、紫色渐变、霓虹感和大面积玻璃拟态。

视觉关键词：

- 职业档案
- 高级编辑器
- 纸张质感
- 安静但有判断力
- 信息清晰、控件克制

颜色：

- 背景：#F5F1E8
- 主文本：#20201D
- 次文本：#706A5F
- 主色：#1F4D3A
- 强调色：#D85C3A
- 辅助色：#C7A76C
- 卡片：#FFFDF7
- 边框：#DED6C7
- 深色区域：#17231D

圆角：

- 卡片 8px
- 按钮 6px
- 输入框 6px

字体：

- 中文：PingFang SC / Microsoft YaHei / system-ui
- 英文数字：Inter / system-ui

## 2. 页面清单

### P1 工作台 Dashboard

用途：用户进入后的主控台。

模块：

- 顶部导航：产品名、资料库、JD 分析、简历、投递、复盘
- 左侧主区域：本周求职进度
- 右侧：快捷操作
- 底部：最近岗位和简历版本

核心组件：

- 数据卡片：投递数、面试数、简历版本、平均匹配度
- 状态看板：待投递、已投递、面试中、Offer
- 快捷按钮：粘贴 JD、生成简历、记录复盘
- 健康度模块：资料完整度、事实依据覆盖、复盘反哺项
- 流程条：导入资料、分析 JD、生成简历、投递复盘
- 下一步行动清单：补资料、导出 PDF、记录复盘

### P1.5 旧简历导入

用途：降低新用户填写资料库的成本。

模块：

- 上传 PDF/Word
- 粘贴旧简历文本
- 解析结果预览
- 待确认信息
- 一键写入资料库

交互：

- 上传后显示解析进度
- 解析结果分为“已识别”和“需要确认”
- 无依据的夸张表述不直接写入资料库

### P2 资料库

用途：管理个人职业素材。

模块：

- 基本信息
- 工作经历
- 项目经历
- 技能标签
- 作品链接

交互：

- 点击经历卡片可编辑
- 每段经历可以添加标签
- 可标记“适合产品岗/技术岗/运营岗”

### P3 JD 分析

用途：粘贴岗位 JD，解析岗位要求。

布局：

- 左侧：JD 输入框
- 右侧：岗位画像
- 下方：匹配度和优化建议

输出：

- 岗位名称
- 核心能力
- 关键词
- 加分项
- 匹配度
- 简历优化建议
- 分项评分：技能、项目、证据
- 缺口提醒：缺少什么数据或案例
- 公司研究卡：业务模式、岗位语境、简历策略
- 岗位成功画像：面试官可能验证的能力

### P4 简历生成器

用途：生成和编辑定制简历。

布局：

- 左侧：简历结构导航
- 中间：简历预览和编辑
- 右侧：AI 建议、关键词覆盖、导出 PDF

交互：

- 点击“重新优化”更新当前模块
- 点击“锁定”防止 AI 改写某段经历
- 点击“导出 PDF”生成简历
- 右侧展示事实依据，说明每条关键表达来自哪段资料库
- 无依据内容进入待补充，不自动写入正式简历
- 导出前检查：事实来源、无编造内容、待确认项

### P4.5 大厂模式

用途：让用户根据大厂/外企/技术岗场景切换简历和面试策略。

模块：

- 大厂 ATS 简洁版
- 外企英文版
- 技术岗深挖版
- 批量岗位分析
- 经验发现问题

交互：

- 点击“大厂 ATS 简洁版”后，系统切换到更适合 ATS 的摘要与结构
- 批量 JD 分析展示共同要求、差异关键词和共用简历底稿

### P5 投递看板

用途：管理投递进度。

状态列：

- 待投递
- 已投递
- 面试中
- Offer
- 结束

卡片字段：

- 公司
- 岗位
- 使用简历版本
- 下一步时间
- 当前备注
- 投递渠道
- 下一步动作

### P6 面试复盘

用途：记录面试并生成改进建议。

模块：

- 面试基本信息
- 被问问题
- 回答表现
- 卡住的问题
- AI 复盘总结
- 下一次准备重点
- 同步到资料库待补充项
- 同步到简历优化建议
- 生成复盘回写队列
- STAR 案例库：Deliver Results、Dive Deep、Earn Trust、Ownership 等

## 3. 墨刀搭建尺寸建议

桌面端画板：

- 1440 x 1024

移动端后续再做：

- 390 x 844

推荐先完成桌面端，因为简历编辑和投递管理更适合大屏。

## 4. 核心页面文案

产品名：

JobFit Resume AI

Slogan：

让每一次投递，都像为这个岗位认真准备过。

主要按钮：

- 粘贴 JD
- 分析岗位
- 生成定制简历
- 导出 PDF
- 记录面试复盘
- 保存为投递记录

空状态文案：

还没有岗位记录。粘贴一个 JD，系统会帮你生成第一版定制简历。

AI 提醒文案：

以下内容基于你的资料库生成，未发现原始依据的内容不会被自动写入简历。

## 5. 组件规范

按钮：

- 主按钮：墨绿色底、米白文字
- 次按钮：透明底、深色文字、浅边框
- 危险/提醒按钮：珊瑚红

卡片：

- 白色纸感背景
- 1px 浅色边框
- 轻微阴影
- 不使用玻璃模糊

标签：

- 技能标签：浅绿底
- 缺口标签：浅珊瑚底
- 加分项：浅金色底

进度条：

- 使用墨绿色主进度
- 不使用蓝紫渐变
